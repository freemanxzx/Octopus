# Octopus MD Sync 浏览器扩展保护与隐私协议 (Privacy Policy)

**最后更新日期：2026年4月15日**

感谢您信任并使用 Octopus MD Sync (Sync Everywhere) 浏览器扩展。我们极度重视您的数字隐私和账号安全。为了透明地向您说明我们如何处理您的数据，特制定本隐私政策，以便您放心使用。

## 1. 核心承诺：绝对的本地化数据处理
Octopus MD Sync 是一款为自媒体创作者提供的跨平台文章发布辅助增强工具。
**本扩展完全基于“本地沙箱中转”架构运行，我们在此庄严承诺：**
*   我们**绝不收集、上传或在任何第三方服务器上存储**您的网页浏览历史。
*   我们**绝不拦截、收集、或外流**您在任何平台（知乎、掘金、微信、CSDN等）的身份验证 Cookie、Session 令牌 或密码。
*   所有的分发注入（DOM Injection）和内容排版动作完全在您本机的当前浏览器内存中处理，随阅随焚。

## 2. 权限声明与使用场景
为了能够将您的 Markdown 或 RichText 文章进行“一键免跳转多平台草稿箱并发”，Chrome Web Store 要求我们申请以下敏感权限，这仅仅是为了在技术层面打通本地网桥，请您知悉其用途：

*   **`host_permissions` *(特权域名权限)*：**
    我们仅仅申请了您明确需要发布的平台域名（如 `*.weixin.qq.com`, `zhuanlan.zhihu.com`, `juejin.cn` 等）。这赋予了插件在后台替您发送“保存草稿”的跨域能力（Bypass CORS），免除您每个网页重复手动粘贴的劳累。
*   **`activeTab` 及 `tabs` *(标签页访问权)*：**
    仅用于在您点击分发时，自动帮您寻找并定位那些已经打开的编辑器标签页，以便触发内置的模拟剪贴板粘贴行为完成注入。
*   **`storage` *(本地存储权)*：**
    仅在您点击发文的一瞬间，用于在桌面端编辑器和 Chrome 扩展之间临时交接您的文章 payload 数据。
*   **`cookies` *(凭证管理权)*：**
    扩展依靠此权限在发送 API 请求时，能够由浏览器引擎自动挂载您的合法凭证（复用您已经登入的状态），这使得您无需向插件提供任何明文密码即可完成授权交互。
