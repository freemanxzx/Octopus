# Octopus MD - 图像处理模块测试用例 (QA Test Cases)

为了保障系统集成的 Web Worker 图片压缩与 Cropper.js 弹窗二次编辑功能在不同的极端场景下的高可靠性和数据流完整性，特制定以下测试用例。

---

## 模块 A：Web Worker 静默图片压缩测试

### 测试用例 A-1：超大型静态原图的无感压缩拦截
*   **前置条件**：确保图床上传通道（如 PicGo 或 Base64 模式）运作正常。
*   **运行步骤**：
    1. 准备一张通过单反相片导出的、大小超过 15MB 且分辨率大于 `4000x3000` 的高清 JPG 图片。
    2. 打开 Octopus MD，在左侧 CodeMirror 编辑器中执行粘贴（`Ctrl/Cmd + V`）或将图片快速拖拽进入编辑器窗口。
    3. 观察粘贴的瞬间，界面输入法光标是否依然连续闪烁，UI 是否能顺滑滚动。
    4. 观察上传生成后的图片 URL。若配置为 Base64 或本地文件，将其下载保存。
*   **验收标准 (Expected Result)**：
    *   主线程无阻塞（光标持续闪烁，浏览器不卡顿弹框）。
    *   导出的图片体积应 **严格小于 500KB**，长宽最大一边应被自动裁切重采样至 **1920px**，同时图片内容依旧清晰可辨，未出现数据损毁（Corrupted Chunk）。

### 测试用例 A-2：微型优化放行策略测试
*   **前置条件**：准备几张只有 100KB 大小的网络配图。
*   **运行步骤**：拖拽至编辑器。
*   **验收标准**：上传速度极快，原始质量完全不被干预。

### 测试用例 A-3：GIF 动态帧隔离保护测试
*   **前置条件**：准备一张 GIF 格式动图（如 3MB 的演示动画）。
*   **运行步骤**：拖拽至编辑器进行上传。
*   **验收标准**：系统应**完全跳过压缩管线**。在右侧预览区渲染后，GIF 图仍保留完整动画帧并能成功进行周期律动无死帧，避免因单一 Canvas 重采样导致其被降维成一张纯静态图片。

## 模块 B：Cropper.js 二次图片画板交互测试

### 测试用例 B-1：图像弹窗的独立事件冒泡激活
*   **前置条件**：在编辑器中先写入 `![测试图](https://dummyimage.com/600x400/000/fff)` 以加载一张远程图片。
*   **运行步骤**：
    1. 在右侧 Markdown 实时渲染预览区（Preview Pane）内，用鼠标单击这段图片。
    2. 再次尝试点击图片周围的文本或空白白板区域。
*   **验收标准**：
    *   精准点击图片时，暗色模态框（CropperModal）应配合动画流畅弹出，中央出现带有九宫格裁剪线的待裁图片。
    *   点击非图形元素时应无任何误判弹出行为。

### 测试用例 B-2：多维度图像矩阵切割测试
*   **前置条件**：成功触发 CropperModal 弹框。
*   **运行步骤**：
    1. 点击 `1:1` 按钮，验证裁剪框是否被严格锁定在了纯正方形比例。
    2. 点击 `向右旋转`，图片是否立刻响应 `+90deg` 顺时针画面重构。
    3. 拖拽四角调整至画面边缘区域，验证框体移动是否如丝般顺滑并拥有边界反弹碰撞保护。
*   **验收标准**：所有工具栏辅助按键对应的 `Cropper.js` 组件实例事件必须触发正常。

### 测试用例 B-3：无缝文字热替换 (String Interpolation) 数据溯回
*   **前置条件**：编辑器内存在多张不同的图片，处于全屏同步工作流当中。
*   **运行步骤**：
    1. 弹起裁剪框，完成任意有效修改，点击下方的 `确认裁剪并替换` 按钮。
    2. 此时观察按钮上是否由于网络耗时浮现了正确的 `Loading Spinner` 状态等待。
    3. 在网络回传绿灯瞬间，关闭弹框。观察左侧 MD 面板中这行图片语句的区别。
*   **验收标准**：
    *   底下的上传逻辑应立刻向配置的云端（如 PicGo）推送修改后的最终切片。
    *   **最核心验收点**：左侧 Markdown 字符串（针对被裁的**原始那一个链接位置**）瞬间自动更替成了**新的 URL** 数据，并在右侧 Preview 端触发了重载。
    *   其他所有的文本结构不发生哪怕一个字节的回车错位。

### 测试用例 B-4：异常防波堤阻断测试
*   **运行步骤**：在上传图床配置被强制写错密码/网络物理断开的状态下点击“确认裁剪并替换”。
*   **验收标准**：应成功在底部冒泡呈现红色的报错（如：`替换失败：Fetch Error XXX`），并且模态框不强行自动销毁、不销毁左侧文本中的历史安全 URL，达成完整的原子性崩溃防御。
