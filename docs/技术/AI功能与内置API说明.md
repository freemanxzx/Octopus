# AI 功能与内置 API 代理说明

## 1. 什么是“内置服务”？
在 MarkPublisher 的 AI 助手与文生图面板中，默认提供了一个名为“内置服务”的选项。这是底层继承自原 `doocs/md` 开源项目保留下来的一个 **API 免费体验池（网关池）**。

## 2. 代码实现原理与架构逻辑
通过追溯前台 `useAIConfigStore` 模块与位于 `packages/shared/src/configs/ai-service-options.ts` 的服务选项预设，其底层核心运作机制如下：

- **网关代理（Proxy Endpoint）**：预设的一个透明代理地址 `https://proxy-ai.doocs.org/v1`。这是由开源社区赞助和建立的类似于 OpenAI 接口兼容格式的中转分发网关。
- **免授权验证（No API Key Required）**：当用户在设置中选择此选项（`default`）后，相关面板**不需要强制用户填入自己私有的 API Key 参数**。当发起对话或绘图时，网络层会把请求通过这个社区跨域网关，自动转发给一些开放了限时免费（Free-tier）或者由社区集中报销的大模型。
- **关联的底层供应模型池**：
  - **文生图生成**：内置服务默认请求由快手开源的极强中文图片大模型 `Kwai-Kolors/Kolors`（可灵引擎），从而实现一键无感绘制文章配图。
  - **发文与润色助手**：主要挂载各家的开源或轻量推理模型（如通义千问 `Qwen2.5-7B-Instruct` 等），用于实现针对用户段落的迅速改写和结构重组。

## 3. 运维与使用建议
该“免钥网络代理墙”的设计初衷，旨在彻底抹平非代码型用户的体验门槛，营造“开箱即用”的极佳用户体验。

但是，如果在日常化运营中存在高度依赖指令响应速度、甚至超深长上下文需求的流水线分发场景，系统建议通过点击侧边栏“⚙️设置”，手动切换配置至诸如“DeepSeek”、“硅基流动”、“OpenAI”或“腾讯”等原生服务商，并注入私有独立申请的 API 密钥，以斩获独立带宽和最优性能。
