# 自媒体工具融合项目 - 项目计划书

> 项目名称：MarkPublisher（暂定）
> 计划日期：2026-04-09
> 版本：v1.0

---

## 一、项目愿景

### 1.1 目标

打造一款覆盖「创作-排版-多平台分发」的一站式自媒体工具，融合 doocs/md 与文颜的核心优势。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **一站式** | 从写稿到发布，一个工具全搞定 |
| **多平台** | 公众号 + 知乎 + 头条 + 掘金 + CSDN + Medium |
| **高效率** | AI 辅助写作 + 精美排版 + 一键分发 |
| **跨端使用** | Web 版 + 桌面客户端（Electron） |

## 三、技术选型

### 3.1 核心技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端框架** | React 18 | 与 doocs/md 一致，生态成熟 |
| **语言** | TypeScript | 类型安全，社区大 |
| **富文本编辑器** | ProseMirror / Slate | doocs/md 已在用 |
| **桌面打包** | **Electron** | 详见下方对比 |
| **状态管理** | Zustand / Redux Toolkit | 轻量且强大 |
| **样式方案** | Tailwind CSS | 开发效率高 |
| **构建工具** | Vite | 快，生态好 |

### 3.2 为什么选择 Electron

| 对比项 | Electron | Tauri |
|--------|----------|-------|
| **包体大小** | 100-150 MB | 2-10 MB |
| **启动速度** | 较慢 | ⚡ 极快 |
| **Web 兼容性** | ✅ 内置 Chromium，一致性好 | ❌ 依赖系统 WebView |
| **npm 生态** | ✅ 完美兼容 | ⚠️ 有限 |
| **富文本编辑器** | ✅ 稳定支持 | ⚠️ 可能有兼容问题 |
| **社区成熟度** | ✅ 成熟稳定 | ⚠️ 相对年轻 |
| **开发效率** | ✅ 高 | ⚠️ Rust + Web 两边调试 |

**结论**：doocs/md 的复杂富文本编辑器需要稳定一致的渲染环境，Electron 是最佳选择。

## 五、开发计划

### 5.1 阶段划分

```
时间线：约 6-9 个月

┌─────────────┐
│  阶段一     │  1-2 个月
│  底层融合   │
└──────┬──────┘
       ↓
┌─────────────┐
│  阶段二     │  2-3 个月
│  功能增强   │
└──────┬──────┘
       ↓
┌─────────────┐
│  阶段三     │  2-3 个月
│  客户端打包 │
└──────┬──────┘
       ↓
┌─────────────┐
│  阶段四     │  持续
│  生态扩展   │
└─────────────┘
```

### 5.2 详细里程碑

#### 阶段一：底层融合（1-2个月）

| 周次 | 任务 | 产出 |
|------|------|------|
| 第1-2周 | 项目初始化、代码仓库搭建 | GitHub 仓库、基础框架 |
| 第3-4周 | 编辑器核心集成（基于 doocs/md） | 可编辑的 Markdown 编辑器 |
| 第5-6周 | 排版引擎集成（主题系统） | 支持 30+ 主题的预览 |
| 第7-8周 | 图床系统整合（13种图床） | 完整的图片上传功能 |

**里程碑**：Web 版 MVP 可运行，支持公众号排版

#### 阶段二：功能增强（2-3个月）

| 周次 | 任务 | 产出 |
|------|------|------|
| 第9-10周 | 多平台适配器开发 | 知乎/头条/掘金格式转换 |
| 第11-12周 | 链接转脚注、样式优化 | 多平台格式正确转换 |
| 第13-14周 | AI 集成（DeepSeek/OpenAI/通义） | AI 辅助写作功能 |

**里程碑**：完整的多平台排版分发工具

#### 阶段三：客户端打包（2-3个月）

| 周次 | 任务 | 产出 |
|------|------|------|
| 第15-16周 | Electron 环境搭建 | 桌面端基础框架 |
| 第17-18周 | 本地存储、离线支持 | 可离线使用的桌面版 |
| 第19-20周 | 系统集成（菜单/通知/自动更新） | 完整的桌面客户端 |
| 第21周 | 测试、Bug 修复、发布 | 正式版发布 |

**里程碑**：Electron 桌面客户端发布

#### 阶段四：生态扩展（持续）

| 任务 | 说明 |
|------|------|
| 主题市场 | 用户可上传/下载主题 |
| 插件系统 | 第三方扩展支持 |
| 社区运营 | 收集反馈，持续迭代 |
